# Pawbie Palette Exploration — 2026-06-16

> Status: Exploration only. Do not treat these as official brand tokens until selected and promoted into `brand-dna.md`.

## Goal

Explore possible Pawbie brand palettes using:

- Pawbie product positioning: fluffy desktop AI pet, companion-first, not a device dashboard
- Existing Pawbie packaging draft: cream package surface, low-contrast lifestyle icons, soft blue information band, kraft strip
- Official Pawbie fur colors: Pink `#FFC5E6`, Cream `#FCFCF7`, Blue `#D8E5F4`
- Adjacent brand references: pet lifestyle, plush toy, reusable/lifestyle goods, cozy home objects

## Initial Recommendation

Use `Packaging Soft Core` as the nearest-term extension of the current brand, then borrow controlled accents from:

- `Plush Boutique` for giftability
- `Calm Tech Companion` for App UI
- `Outdoor Picnic` for diary/life observation content

Do not promote `Toy Shelf Pop` wholesale into the core brand yet. It is useful for campaigns, but it risks making Pawbie look more like a children's toy than a companion product.

## Current Shortlist Notes

Based on review of the generated moodboards, the most promising directions are:

### 10. Neutral Clay Charcoal

Strongest core-system candidate so far. It feels more like a design object and lets Pawbie's eyes/fur colors become emotional accents instead of making the whole brand pastel.

Use for:
- App shell and primary UI surfaces
- Brand documentation
- Higher-end product pages
- Packaging structure and instruction cards

### 09. Butter Cornflower

Good bright energy, but the colorful area should be smaller. Treat it as a campaign/accent direction rather than the permanent core system.

Use for:
- launch banners
- social cards
- badges
- small color-block moments

Avoid:
- large App surfaces
- full packaging backgrounds

### 07. Terracotta Cocoa

Promising warm Maillard/lifestyle direction. The red/cocoa parts should be smaller so it does not become too heavy.

Use for:
- kraft strips
- typography accents
- side panels
- package structure
- editorial cards

### 04. Collage Grammar

The illustration palette feels comfortable, but it raises the question: how should a many-color system work in Web/App?

Recommendation:
- Use the full palette inside illustrations, stickers, patterns, campaign modules, and editorial assets.
- Keep App chrome restrained: warm surface + graphite/neutral UI + one selected-state color.
- On Web, reserve many-color moments for hero art, product modules, section dividers, and social/campaign blocks.

### 01. Packaging Blue

Closest to current packaging, but it risks feeling like a baby/infant cotton brand if used too softly.

To mature it:
- add stronger graphite/black typography
- increase scale of type and layout confidence
- pair with kraft, clay, terracotta, or olive accents
- reduce large pale-blue/pale-pink surface area

## Revised Shortlist Moodboards

Four revised boards were generated from the current shortlist feedback and added to the HTML board:

- `09r Butter Cornflower`: keeps brightness but reduces colorful area to small tabs, dots, labels, and UI accents.
- `07r Maillard Reduced Red`: keeps the warm lifestyle direction while shrinking red into small signals rather than large fields.
- `04r Controlled Multicolor UI`: demonstrates the system rule for many colors: neutral UI shell first, rich colors only in illustration, sticker, badge, and pattern modules.
- `01r Packaging Matured`: stays close to the packaging palette but adds graphite, kraft, and stronger layout structure to avoid baby/infant cotton cues.

Working principle: the official system should not choose between "minimal" and "colorful." It can be minimal at the UI/system layer and colorful at the brand-world layer.
