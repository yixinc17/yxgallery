Apple-style App Color System

按 iOS 的颜色逻辑使用 Pawbie 品牌色

Apple 的 app color system 不是把品牌色铺满界面，而是把颜色拆成 background、surface、tint、semantic state、content identity 和 accessibility 几层。Pawbie 应该学习这种分层：系统色负责可用性，品牌色负责识别和情绪，危险色只负责风险。

Apple HIG · Color Dark Mode Accessibility

1. Background first

先定义系统背景，而不是先选主色。iOS 界面通常让背景承担安静的空间秩序。

  • 全局底色用 `color.canvas.softCream`。
  • 不要给 Home / Diary / Settings 各自发明不同大背景。
  • 顶部和底部导航尽量透明或同底色，让内容呼吸。

2. Surface creates hierarchy

像 iOS 的 grouped table 和 sheet 一样，用 surface 层级表达结构，少用重阴影和大色块。

  • `shellWhite` 用于 sheet、dialog、settings row。
  • `warmMilk` 用于 Home hero、empty state、onboarding illustration area。
  • `paleSand` 用于 selected pill、control base、低强调模块。

3. Tint is interaction, not decoration

iOS 的 tint color 通常告诉用户“这里可以操作 / 这里被选中”。Pawbie 的 tint 要小、准、可预测。

  • `tint.primary` 只用于 selected date、progress、active dot、小 CTA。
  • 按钮不需要全部 kraft；强操作可用 `#4A332C` 深色按钮。
  • 不要把 `tint.primary` 用作整页 header、tab bar 或大背景。

4. Semantic colors stay semantic

系统状态色不能被品牌情绪污染。危险色一旦变成装饰，用户就会失去对风险的敏感度。

  • `danger.signal` 只用于 remove、error、failed update、irreversible action。
  • 普通提醒、轻微 warning 可先用 copy、icon、位置和 `tint.soft`。
  • 删除确认可用 `danger.softBg`，但不要整屏红。

5. Brand color lives in content

Apple app 里品牌识别往往在内容、插画、icon、状态表达中出现，而不是把 navigation chrome 染满。

  • Pink / Cream / Blue 是 Pawbie 身份色，不是 app selected color。
  • 毛色用于 Pawbie 本体、头像、设备卡、贴纸、mood 插画。
  • 三只同时出现时固定 Pink / Cream / Blue，避免系统感漂移。

6. Dark mode is a separate palette

不要把浅色模式自动反相。Pawbie 的暗色模式应该像温暖夜间房间，而不是纯黑科技界面。

  • Dark canvas 用 `#1F1916`，surface 用 `#2A211D` / `#3A2D27`。
  • Dark tint 用更亮的 `#D1A06F` 保持可见。
  • Diary 阅读优先，mood 色只做小面积辅助。

7. Accessibility decides final usage

颜色漂亮不等于能上线。对比度、动态文字、色盲可辨识和 Reduce Transparency 都会影响最终选择。

  • 正文和小字最低按 4.5:1 检查。
  • selected state 必须同时有 pill、dot、字重或 icon 状态。
  • 低对比品牌色只能做背景 tint 或插画，不承载关键文本。

8. Materials stay matte

即使参考 iOS，也不需要追 Liquid Glass 或强透明材质。Pawbie 的品牌触感来自纸面、毛绒和柔软包装。

  • 优先 1px warm border、浅 surface 差异和哑光 fill。
  • 透明 blur 只用于必要 overlay，不作为品牌主视觉。
  • yellow 只做快乐点缀，不做系统主按钮。

Updated Result

Pawbie Warm Matte System is now the latest baseline. 07r Maillard Reduced Red remains the historical route that led here, but app screens now use the refined token set above: softer foundation, warmer primary text, smaller interaction tint, and clearer danger red.

App Color Rule

Use about 70% warm neutral UI base, 20% text/structure/small tint, 5–15% Pawbie identity colors, and only tiny semantic signals. Brand colors should support recognition without becoming the whole interface.

Key Color Hypothesis

The app key color should not default to any fur color. Pawbie has pink, cream, and blue product variants, so interaction color sits outside the fur trio: `#A87545` for small active states and `#4A332C` for readable structure.
